## Overview

The `UsersController` handles all user management functionality including creating, editing, and deleting users, managing permissions through ACL (Access Control Lists), handling authentication, and generating PDF reports. It supports multiple user types including administrators and vendors.

**Models Used**: `User`, `Group`, `Aco`, `Aro`, `ArosAco`, `Locale`

**Helpers**: `Html`, `Form`, `Javascript`

## ACL Integration

The Users controller integrates deeply with CakePHP's ACL system:

**ARO Structure**:
- Groups are parent AROs
- Users are child AROs under their group
- Changing a user's group updates the ARO parent_id

**Permission Management**:
- Permissions are managed per-user or per-group
- ACO tree represents controllers and actions
- The `permission()` action provides a UI for managing permissions
- AJAX updates allow/deny permissions in real-time

## Database Structure

The controller works with the `users` table:
- `id_usuario`: Primary key
- `username`: Login username
- `password`: Hashed password
- `email_usuario`: Email address
- `perfil_usuario`: Display name/profile
- `groups_idgrupos`: Foreign key to groups
- `locale_id_local`: Associated business location (for vendors)
