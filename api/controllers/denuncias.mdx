## Overview

The `DenunciasController` manages the complaint and report system in Yoneily. It handles both user complaints about products/services and business responses to those complaints. The controller supports complaints for both gallery items and promotions, with a complete workflow from submission to response.

**Models Used**: `Denuncia`, `Locale`, `Gallery`, `User`

**Helpers**: `Html`, `Form`, `Javascript`

## Authentication & Access Control

```php
function beforeFilter() {
    $this->Auth->allowedActions = array(
        'add', 'consulta', 'denunciar', 'fin_denunciar', 
        'denunciar_promo', 'registro_reclamo', 'registro_denuncia',
        'detallemensajepromo'
    );
}
```

The following actions are publicly accessible without authentication:
- add
- consulta
- denunciar
- fin_denunciar
- denunciar_promo
- registro_reclamo
- registro_denuncia
- detallemensajepromo

## Usage Examples

### Submitting a Complaint

```php
// POST to /denuncias/denunciar
$data = [
    'Denuncia' => [
        'mensaje' => 'Product was damaged',
        'galeria_id_galeria' => 123,
        'register_idregistro' => 456,
        'locale_id_local' => 789
    ]
];
```

### Responding to a Complaint (Admin)

```php
// POST to /denuncias/registro_denuncia
$data = [
    'Denuncia' => [
        'usuario' => 1,
        'id_respuesta' => 50,
        'mensaje' => 'We apologize for the inconvenience...',
        'galeria_id_galeria' => 123,
        'local' => 789
    ]
];
```

## Database Structure

The controller works with the `denuncias` table with these key fields:
- `id_denuncia`: Primary key
- `mensaje`: Complaint message
- `fecha`: Date of complaint
- `resp`: Response user ID (0 = pending)
- `status`: Read status (0 = unread, 1 = read)
- `register_idregistro`: Customer ID
- `galeria_id_galeria`: Gallery product ID (optional)
- `promocion_id_promo`: Promotion ID (optional)
- `locale_id_local`: Business location ID
- `nombre`: Responder name
