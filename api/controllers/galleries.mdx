## Overview

The `GalleriesController` handles product gallery management including creating, editing, and displaying product images. It integrates image processing using the SimpleImage library to generate thumbnails and resized versions. The controller supports inventory tracking and business-specific product filtering.

**Models Used**: `Gallery`, `Archivo`, `Category`, `Locale`

**Helpers**: `Html`, `Form`, `Javascript`

**External Libraries**: `SimpleImage.php` (vendor library for image processing)

## Image Processing

The controller automatically generates two image versions for each uploaded gallery image:

1. **Thumbnails**: 124x124px (stored in `/files/galeria/thumbnails/`)
2. **Normal**: 600x480px (stored in `/files/galeria/normal/`)
3. **Original**: Full size (stored in `/files/galeria/otras/`)

### add()

Creates a new gallery/product with image upload and processing.

<ParamField path="layout" type="string" default="admin">
  Uses admin layout
</ParamField>

<ParamField body="this->data['Gallery']" type="array" required>
  Gallery data including:
  - locale_id_local: Business name (converted to ID)
  - nombre_galeria: Gallery name
  - descripcion_galeria: Description
  - precio: Price
  - cantidad: Quantity available
  - thumbnails: Uploaded image file
</ParamField>

**View Variables**:
- `empresa`: Business name (if user has locale_id_local)
- `users`: User list
- `archivos`: File list

**Behavior**:
1. Validates business exists and is active (status = 1)
2. Converts business name to locale ID
3. Sets initial inventory values:
   - `cantidad_existente` = cantidad
   - `prod_vendidos` = 0
4. Saves gallery record
5. Processes uploaded image:
   - Creates 124x124 thumbnail
   - Creates 600x480 normal size
   - Maintains aspect ratio
6. Shows success/error flash message

**Image Processing**:
```php
$image = new SimpleImage();
$image->load($original_file_path);
if($image->getWidth() > 124){
    $image->resizeToWidth(124);
}
if($image->getHeight() > 124){
    $image->resizeToHeight(124);
}
$image->save($conversion);
```

### deleteimg($id, $idfile)

Deletes a specific image file associated with a gallery.

<ParamField path="id" type="integer" required>
  Gallery ID
</ParamField>

<ParamField path="idfile" type="integer" required>
  File/Image ID to delete
</ParamField>

**Behavior**:
- Uses raw SQL query with INNER JOINs
- Deletes from archivos_galleries and archivos tables
- Shows success/error flash message
- Redirects to index

**SQL Query**:
```php
DELETE FROM archivos_galleries, archivos 
USING archivos_galleries 
INNER JOIN archivos ON archivos_galleries.files_id_file=archivos.id_file 
INNER JOIN galleries ON archivos_galleries.galerias_id_galeria=galleries.id_galeria 
WHERE galleries.id_galeria = $id AND archivos.id_file = $idfile
```

**Access**: Requires authentication

## Image Processing Details

### Thumbnail Generation

The controller creates thumbnails with these specifications:
- **Max width**: 124px
- **Max height**: 124px
- **Maintains aspect ratio**: Yes
- **Storage path**: `/webroot/files/galeria/thumbnails/`

### Normal Size Generation

Normal-sized images for display:
- **Max width**: 600px
- **Max height**: 480px
- **Maintains aspect ratio**: Yes
- **Storage path**: `/webroot/files/galeria/normal/`

### Image Processing Flow

1. Original image uploaded to `/files/galeria/otras/`
2. SimpleImage library loads original
3. Resize to thumbnail dimensions if needed
4. Save thumbnail to thumbnails directory
5. Reload original for normal size
6. Resize to normal dimensions if needed
7. Save normal size to normal directory

## Database Structure

The controller works with the `galleries` table:
- `id_galeria`: Primary key
- `nombre_galeria`: Product name
- `descripcion_galeria`: Product description
- `precio`: Price
- `cantidad`: Original quantity
- `cantidad_existente`: Current stock
- `prod_vendidos`: Number sold
- `thumbnails`: Image filename
- `locale_id_local`: Business location ID
- `publicar`: Published status (0=unpublished, 1=published)
- `fechacre_galeria`: Creation date
- `user_id`: Creator user ID

## Inventory Management

When a gallery is created or edited:
- `cantidad_existente` is set to the initial `cantidad`
- `prod_vendidos` is initialized to 0
- These values are updated elsewhere (e.g., Ventas controller) when sales occur
