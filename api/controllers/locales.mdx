## Overview

The `LocalesController` manages business locations (locales) in the Yoneily system. It handles venue registration, profile management, image processing for business logos and location maps, and business activation/deactivation. Each locale receives a unique access code for vendor registration.

**Models Used**: `Locale`, `Archivo`, `Category`, `User`, `Group`, `Aco`, `Aro`, `ArosAco`

**Helpers**: `Html`, `Form`, `Javascript`

**External Libraries**: `SimpleImage.php` (vendor library for image processing)

## Authentication & Access Control

```php
function beforeFilter(){
    $this->Auth->allow('acceso');
}
```

Only the `acceso` action is publicly accessible. All other actions require authentication.

## Image Processing

The controller processes two types of images for each locale:

1. **Business Logo** (`nombre_file`):
   - Thumbnail: 96x72px (stored in `/files/locales/thumbnails/`)
   - Slider: 600x480px (stored in `/files/locales/slider/`)

2. **Location Map** (`ubicacion_file`):
   - Slider size: 600x480px (stored in `/files/locales/slider/`)

### add()

Registers a new business location with automatic code generation and image processing.

<ParamField path="layout" type="string" default="admin">
  Uses admin layout
</ParamField>

<ParamField body="this->data['Locale']" type="array" required>
  Locale data including:
  - rif: Tax ID (must be unique)
  - nombre_empresa: Business name
  - encargado_nombre: Manager first name
  - encargado_apellido: Manager last name
  - correo: Email address
  - telefono: Phone number
  - direccion: Address
  - nombre_file: Business logo (uploaded file)
  - ubicacion_file: Location map (uploaded file)
</ParamField>

**View Variables**:
- `users`: User list

**Behavior**:
1. Generates unique 6-character access code using MD5
2. Validates RIF uniqueness
3. Saves locale record
4. Processes business logo:
   - Creates 96x72 thumbnail
   - Creates 600x480 slider image
5. Processes location map:
   - Creates 600x480 slider image
6. Shows success/error flash message

**Access Code Generation**:
```php
$cadena = "abcdefghijklmnopqrstuvwxyz";
$cadena = md5($cadena);
srand((double)microtime()*1000000);
$inicio = rand(0,27);
$codigo = substr($cadena,$inicio,6);
$this->data['Locale']['codigo'] = $codigo;
```

**Validation**:
- Checks if RIF already exists in database
- If RIF exists, shows error and doesn't save

**Access**: Requires authentication

## Access Code System

Each locale receives a unique 6-character code upon registration:

**Generation Method**:
1. Start with alphabet string
2. Hash with MD5
3. Use microsecond-seeded random number
4. Extract 6 characters from random position
5. Store in `Locale.codigo` field

**Usage**:
- Vendors use this code during registration
- Links vendor accounts to business locations
- Referenced in UsersController::add_vendedor()
- Can be validated via UsersController::consulta_codigo()

## Database Structure

The controller works with the `locales` table:
- `id_local`: Primary key
- `rif`: Tax ID (unique)
- `nombre_empresa`: Business name
- `encargado_nombre`: Manager first name
- `encargado_apellido`: Manager last name
- `correo`: Email address
- `password`: Password (plain text - security issue)
- `telefono`: Phone number
- `direccion`: Address
- `nombre_file`: Logo filename
- `ubicacion_file`: Location map filename
- `codigo`: Access code for vendor registration
- `status`: Active status (0=inactive, 1=active)

## Security Considerations

**Warning**: The `acceso()` function has critical security vulnerabilities:

1. **Plain Text Passwords**: Passwords stored and compared without hashing
2. **No Session Management**: Doesn't properly establish authenticated session
3. **No CSRF Protection**: No token validation
4. **Incomplete Implementation**: Redirects without setting session variables

**Recommendation**: This function should be refactored to use CakePHP's Auth component with proper password hashing (Security::hash or bcrypt).
