## Overview

The `PromosController` handles promotional product management in the Yoneily system. It provides functionality similar to the Galleries controller but specifically for promotional items and special offers. The controller includes image processing, inventory tracking, and business-specific filtering.

**Models Used**: `Promo`, `Archivo`, `Category`, `Locale`

**Helpers**: `Html`, `Form`, `Javascript`

**External Libraries**: `SimpleImage.php` (vendor library for image processing)

## Image Processing

The controller automatically generates two image versions for each promotional image:

1. **Thumbnails**: 450x280px (stored in `/files/promos/thumbnails/`)
2. **Slider**: 779x280px (stored in `/files/promos/slider/`)
3. **Original**: Full size (stored in `/files/promos/otras/`)

### add()

Creates a new promotional item with image upload and processing.

<ParamField path="layout" type="string" default="admin">
  Uses admin layout
</ParamField>

<ParamField body="this->data['Promo']" type="array" required>
  Promo data including:
  - locale_id_local: Business name (converted to ID)
  - nombre_promo: Promotion name
  - descripcion_promo: Description
  - precio: Price
  - cantidad: Quantity available
  - fecha_inicio: Start date
  - fecha_fin: End date
  - thumbnails: Uploaded image file
</ParamField>

**View Variables**:
- `empresa`: Business name (if user has locale_id_local)
- `users`: User list
- `archivos`: File list

**Behavior**:
1. Validates business exists and is active (status = 1)
2. Converts business name to locale ID
3. Sets initial inventory values:
   - `cantidad_existente` = cantidad
   - `prod_vendidos` = 0
4. Saves promo record
5. Processes uploaded image:
   - Creates 450x280 thumbnail
   - Creates 779x280 slider image
   - Maintains aspect ratio
6. Shows success/error flash message

**Image Processing**:

// Slider processing

    $image->resizeToWidth(779);
}
if($image->getHeight() > 280){
    $image->resizeToHeight(280);

## Image Processing Details

### Thumbnail Generation

Promotional thumbnails are larger than gallery thumbnails:
- **Max width**: 450px
- **Max height**: 280px
- **Maintains aspect ratio**: Yes
- **Storage path**: `/webroot/files/promos/thumbnails/`
- **Use case**: Grid displays, preview cards

### Slider Generation

Wide banner-style images for promotional sliders:
- **Max width**: 779px
- **Max height**: 280px
- **Maintains aspect ratio**: Yes
- **Storage path**: `/webroot/files/promos/slider/`
- **Use case**: Homepage banners, promotional carousels

### Image Processing Flow

1. Original image uploaded to `/files/promos/otras/`
2. SimpleImage library loads original
3. Resize to thumbnail dimensions (450x280) if needed
4. Save thumbnail to thumbnails directory
5. Reload original for slider size
6. Resize to slider dimensions (779x280) if needed
7. Save slider image to slider directory

## Database Structure

The controller works with the `promos` table:
- `id_promo`: Primary key
- `nombre_promo`: Promotion name
- `descripcion_promo`: Promotion description
- `precio`: Promotional price
- `cantidad`: Original quantity
- `cantidad_existente`: Current stock
- `prod_vendidos`: Number sold
- `fecha_inicio`: Start date
- `fecha_fin`: End date
- `thumbnails`: Image filename
- `locale_id_local`: Business location ID
- `publicar`: Published status (0=unpublished, 1=published)
- `fechacre_promo`: Creation date
- `user_id`: Creator user ID

## Inventory Management

Similar to galleries:
- `cantidad_existente` is set to the initial `cantidad` on creation
- `prod_vendidos` is initialized to 0
- Both values reset to initial state on edit (which may not be desired behavior)
- Values updated by sales system when promo items are purchased

## Known Issues

1. **Edit Function Bug**: The `edit()` function calls `create()` before saving, which may create duplicates instead of updating
2. **DeleteImg Table Reference**: The `deleteimg()` function references wrong table (galleries instead of promos)
3. **Inventory Reset on Edit**: Editing a promo resets `prod_vendidos` to 0 and `cantidad_existente` to `cantidad`, losing sales history
4. **Image Reprocessing**: Every edit reprocesses images even if image wasn't changed

## Best Practices

### Creating Time-Limited Promotions

```php
$data = [
    'Promo' => [
        'nombre_promo' => 'Weekend Special',
        'fecha_inicio' => date('Y-m-d'), // Today
        'fecha_fin' => date('Y-m-d', strtotime('+3 days')), // 3 days from now
        'cantidad' => 100,
        // ... other fields
    ]
];
```

### Optimal Image Dimensions

For best results, upload promotional images with:
- **Aspect ratio**: ~2.78:1 (e.g., 779x280)
- **Minimum dimensions**: 779x280px
- **Format**: JPG or PNG
- **File size**: < 2MB for faster processing
