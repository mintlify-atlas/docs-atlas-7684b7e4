## System requirements

Before installing Yoneily, ensure your server meets the following requirements:

### Required software

- **PHP**: Version 5.3.5 or higher (PHP 5.x recommended for CakePHP 1.3 compatibility)
- **MySQL**: Version 5.5.8 or higher
- **Apache**: Version 2.x with mod_rewrite enabled
- **Web server**: Apache with .htaccess support

### PHP extensions

Ensure the following PHP extensions are installed and enabled:

- `mysql` or `mysqli` - MySQL database connectivity
- `mbstring` - Multi-byte string handling
- `session` - Session management
- `json` - JSON encoding/decoding

<Warning>
CakePHP 1.3 requires PHP 5.x. Do not use PHP 8.x as it will cause compatibility issues. Use PHP 5.6 or PHP 7.4 (with compatibility mode) maximum.
</Warning>

## Installation steps

<Steps>
  <Step title="Download the source code">
    Extract the Yoneily system files to your web server directory.

    ```bash
    cd /var/www/html
    # Extract your sistema_magdaleno files here
    ```
  </Step>

  <Step title="Configure Apache mod_rewrite">
    Verify that Apache mod_rewrite is enabled:

    ```bash
    # On Ubuntu/Debian
    sudo a2enmod rewrite
    sudo systemctl restart apache2

    # On CentOS/RHEL
    # mod_rewrite is typically enabled by default
    sudo systemctl restart httpd
    ```

    The system includes three `.htaccess` files for URL rewriting:

    **Root .htaccess** (`/.htaccess`):
    ```apache
    <IfModule mod_rewrite.c>
       RewriteEngine on
       RewriteRule    ^$ app/webroot/    [L]
       RewriteRule    (.*) app/webroot/$1 [L]
    </IfModule>
    ```

    Ensure your Apache virtual host allows `.htaccess` overrides:

    ```apache
    <Directory /var/www/html/sistema_magdaleno>
        AllowOverride All
        Require all granted
    </Directory>
    ```
  </Step>

  <Step title="Set file permissions">
    The `app/tmp` directory and its subdirectories must be writable by the web server:

    ```bash
    cd sistema_magdaleno
    chmod -R 777 app/tmp
    chmod -R 777 app/tmp/cache
    chmod -R 777 app/tmp/logs
    chmod -R 777 app/tmp/sessions
    ```

    For production environments, use more restrictive permissions:

    ```bash
    # Set ownership to web server user
    chown -R www-data:www-data app/tmp
    chmod -R 775 app/tmp
    ```
  </Step>

  <Step title="Create the MySQL database">
    Create a new MySQL database for the system:

    ```sql
    CREATE DATABASE magdaleno DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci;
    ```

    Create a MySQL user with appropriate privileges:

    ```sql
    CREATE USER 'yoneily_user'@'localhost' IDENTIFIED BY 'your_secure_password';
    GRANT ALL PRIVILEGES ON magdaleno.* TO 'yoneily_user'@'localhost';
    FLUSH PRIVILEGES;
    ```
  </Step>

  <Step title="Import the database schema">
    Import the provided SQL file to set up the database structure:

    ```bash
    mysql -u yoneily_user -p magdaleno < localhost.sql
    ```

    The SQL file creates the following key tables:
    - `acos` - Access Control Objects for permissions
    - `aros` - Access Request Objects for users/groups
    - `users` - User accounts
    - `denuncias` - Complaint records
    - `locales` - Store/location information
    - `galleries` - Media and promotional content
  </Step>

  <Step title="Configure database connection">
    Edit the database configuration file at `app/config/database.php`:

    ```php
    <?php
    class DATABASE_CONFIG {
        var $default = array(
            'driver' => 'mysql',
            'persistent' => false,
            'host' => 'localhost',
            'login' => 'yoneily_user',
            'password' => 'your_secure_password',
            'database' => 'magdaleno',
            'prefix' => '',
            'encoding' => 'utf8',
        );
    }
    ?>
    ```

    <Note>
    The original configuration uses encoding `'utf-8'` but CakePHP expects `'utf8'` (without the dash). Update this if you encounter encoding issues.
    </Note>
  </Step>

  <Step title="Verify installation">
    Access the system through your web browser:

    ```
    http://localhost/sistema_magdaleno
    ```

    If everything is configured correctly, you should see the application homepage or login screen.
  </Step>
</Steps>

## Post-installation checklist

After completing the installation, verify the following:

- [ ] Web server displays the application without 404 errors
- [ ] Database connection is successful (no database errors)
- [ ] Login functionality works correctly
- [ ] File uploads work (check `app/webroot/files` permissions)
- [ ] Session management is functional
- [ ] Clean URLs work without `index.php` in the path

## Troubleshooting

### Issue: 500 Internal Server Error

**Solution**: Check Apache error logs and verify:
- `.htaccess` files are present in root, `app/`, and `app/webroot/`
- `AllowOverride All` is set in Apache configuration
- mod_rewrite is enabled

### Issue: Database connection failed

**Solution**: Verify in `app/config/database.php`:
- Database name matches the created database
- Username and password are correct
- MySQL server is running
- Host is correct (usually `localhost`)

### Issue: Blank page or white screen

**Solution**: Enable debug mode in `app/config/core.php:36`:

```php
Configure::write('debug', 2);
```

This will display detailed error messages. Remember to set it back to `0` in production.

### Issue: tmp directory not writable

**Solution**: Ensure proper permissions:

```bash
chmod -R 777 app/tmp
chmod -R 777 app/tmp/cache
chmod -R 777 app/tmp/logs
chmod -R 777 app/tmp/sessions
```

<Warning>
Never deploy to production with `Configure::write('debug', 2)` enabled. This exposes sensitive system information and SQL queries to users.
</Warning>

## Docker installation (alternative)

For development environments, you can use Docker:

```dockerfile
FROM php:5.6-apache

RUN docker-php-ext-install mysql mysqli
RUN a2enmod rewrite

COPY sistema_magdaleno/ /var/www/html/
RUN chown -R www-data:www-data /var/www/html/app/tmp

EXPOSE 80
```
